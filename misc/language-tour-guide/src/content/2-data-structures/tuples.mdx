# Tuples - Fixed Size Containers

Tuples are ordered collections with a **fixed number of elements**. They're stored contiguously in memory, making element access by index very fast (O(1)).

**Characteristics:**

- **Fixed size**: Can't add/remove elements
- **O(1) access**: Random access by index
- **Immutable**: Updates create new tuples
- **Pattern matching**: Great for structured returns

**Common uses:**

- Function returns (especially `{:ok, value} / {:error, reason})`
- Grouping related values
- Fixed-size coordinates or points
- Database rows

empty = {}
point = {10, 20}

error = {:error, "Not found"}

# Pattern matching - the Elixir way
case {:ok, "data"} do
  {:ok, value} -> IO.puts("Success: #{value}")
  {:error, msg} -> IO.puts("Error: #{msg}")
end

# Updating tuples (creates new tuple)

  end
