# Pipe Operator

The pipe operator `|>` is one of Elixir's most distinctive features.

It takes the result of the expression on the left and passes it as the first argument to the function on the right.

This allows you to write data transformations in a clear, readable way.

export const defaultCode = `# Without pipes (nested function calls)
result1 = String.upcase(String.trim(" hello world "))
IO.puts("Without pipes: #{result1}")

# With pipes (much more readable!)

result2 = " hello world "
|> String.trim()
|> String.upcase()
IO.puts("With pipes: #{result2}")

# More complex example

numbers = [1, 2, 3, 4, 5]
|> Enum.map(fn x -> x \* 2 end)
|> Enum.filter(fn x -> x > 5 end)
|> Enum.sum()

IO.puts("Sum of doubled numbers > 5: #{numbers}")

# Custom functions in pipes

defmodule PipeExample do
  def add(x, y), do: x + y
  def multiply(x, y), do: x \* y
end

result3 = 10
|> PipeExample.add(5)
|> PipeExample.multiply(2)
IO.puts("(10 + 5) \* 2 = #{result3}")`
